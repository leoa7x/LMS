# Decisiones sobre simuladores

## Regla de decision

Todo simulador debe evaluarse primero por viabilidad tecnica y valor pedagogico.

## Tipos soportados

- Embebible existente
- Envoltura o adaptacion de tercero
- Propio basico
- Propio avanzado

## Criterios para embebido

- Puede integrarse dentro del portal sin romper la experiencia
- Permite captura de progreso o al menos eventos minimos
- No exige abrir una aplicacion externa para el estudiante
- Cumple con licenciamiento del proyecto

## Criterios para propio

- No existe alternativa embebible viable
- El tercero no cumple los requisitos funcionales
- El seguimiento academico requerido necesita control total
- Se requiere simulacion especifica de fallas, diagramas o componentes reales

## Estrategia

Fase inicial:

- preparar el modulo
- modelar metadata y sesiones
- soportar integracion embebida y nativa

Fase posterior:

- priorizar simuladores con mayor impacto academico
- clasificar por complejidad 2D, 3D, guided flow o event-driven
