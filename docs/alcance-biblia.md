# Alcance Biblia del Proyecto

## Regla principal

El contenido de `texto_extraido.md` y su PDF fuente asociado es la biblia funcional y academica del proyecto LMS.

No se toma como referencia secundaria. Es el alcance base que el sistema debe cumplir.

## Lo que se considera obligatorio

- Portal web para profesores y estudiantes
- Acceso 24/7
- Accesos simultaneos ilimitados
- Acceso por 36 meses
- Gestion de contenidos por nivel o asignacion
- Seguimiento individual del estudiante
- Evaluaciones pre-curso y post-curso
- Reasignacion de pruebas por parte del profesor
- Cuestionarios por modulo
- Material didactico centralizado en la misma plataforma
- Glosario tecnico integrado
- Impresion o salida PDF
- Contenido en espanol e ingles
- Sin llaves de software o hardware
- Sin necesidad de servidores on-premise en IPT Veraguas o MEDUCA
- Simuladores integrados al flujo de aprendizaje
- Material interactivo con e-books, videos, fotografias y emulaciones 3D
- Integracion con contenido orientado a certificaciones industriales

## Regla sobre simuladores

El proyecto debe contemplar dos enfoques validos:

1. Simuladores embebidos
2. Simuladores propios

Si una solucion embebible no existe, no cumple, o no es viable tecnicamente, se puede desarrollar una solucion propia.

## Cobertura academica obligatoria

El LMS debe cubrir como minimo las areas y contenidos indicados en el documento rector, incluyendo:

- Automatizacion
- Electronica
- Electricidad
- Potencia de fluidos
- Tecnologia verde
- Manufactura ajustada
- Mecanizado
- Procesos de manufactura
- Materiales
- Mecanica
- Control de procesos
- Aseguramiento de calidad
- Seguridad
- Termico
- Industria 4.0
- Autotronica
- Vehiculos hibridos
- Vehiculos electricos

Tambien deben contemplarse:

- Cursos que obligatoriamente usan simuladores virtuales
- Cursos preconfigurados por tema
- Modulos de autotronica, vehiculos hibridos y vehiculos electricos

## Criterio de implementacion

Cualquier decision de arquitectura, UX, contenido, backlog o desarrollo debe justificarse contra este alcance.

Si algo contradice este documento rector, prevalece el documento rector.
